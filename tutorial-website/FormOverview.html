What is a form

It is an element that allows user to input data that is sent to a server for processing. Users fill out the forms using checkboxes, radio buttons, or text fields for example.

The <input> Element

The <input> element is the most important form element. The <input> element has many variations, depending on the type attribute. Here are the types of attribute:

Type Description
text Defines normal text input
radio Defines radio button input (for selecting one of many choices)
submit Defines a submit button (for submitting the form)

Form tags

Tag Description
<form> makes a form for user imput
<input> for user input
<textarea> multiline input control
<button> adds a clickable button
<select> adds a dropdown list
<optgroup> defines a group of related options in a dropdown list
<option> adds an option to the dropdown list
<label> defines a label in the <input> element
<datalist> Specifies a list of pre-defined option for input controls
<Keygen> Defines a Key-Pair generator field
<output> Defines the result of Calculation

GET and POST

GET and POST are HTTP Request Methods:

  • GET - Requests data from a specified resource
  • POST - Submits data to be processed to a specified resource

Features of GET requests:

  • Can be cached
  • Remainemain in the browser history
  • Can be bookmarked
  • Should never be used when dealing with sensitive data
  • Have length restrictions
  • Should be used only to retrieve data

Features of POST requests:

  • Are never cached
  • Do not remain in the browser history
  • Cannot be bookmarked
  • Have no restrictions on data length
